# SQL 1ヶ月マスター学習カリキュラム

各週とも**座学**と**ハンズオン**の両方を行う構成です。`SQL実装の言語化.md` は補助教材として、ハンズオンと合わせて参照します。

## 学習の進め方

- **1週あたりの目安**: 2〜3時間（複数回に分けてOK）
- **環境**: Supabase（PostgreSQL）※セットアップ済みが前提
- **教材**: `sql-practice.md` をメインに、`SQL実装の言語化.md` を補助的に使用

---

## 第1週：基礎の土台を固める

### 目標
- SQLを書く前の思考整理の流れを理解する
- SELECT、WHERE、ORDER BY、LIMIT を使いこなせるようになる

### 座学（`SQL実装の言語化.md` を参照）
- 「1. 用語の説明」「2. SQLを書く前の思考整理の手順」
- ディメンション（分析の切り口）とメジャー（出したい数値）の違い
- 集約したメジャーを出すかどうかの判断
- 出力する単位・rawデータ・出力カラムの整理の仕方

### ハンズオン
- 演習1：SELECT 基本（1-1, 1-2）
- 演習2：WHERE での抽出（2-1, 2-2）
- 演習3：ORDER BY / LIMIT（3-1, 3-2）
- 演習1〜3を解答を見ずに再度書く。各クエリの「ディメンションとメジャーは何か？」を自分で説明してみる

**ポイント**: 演習1〜3は集約不要なので GROUP BY は使わない。クエリを書く前に「何を1行として出すか」を整理してから書く。

## 第2週：集計と結合をマスターする

### 目標
- GROUP BY と HAVING を使った集計が書けるようになる
- JOIN で複数テーブルを結合し、必要な情報を取り出せるようになる

### 座学（`SQL実装の言語化.md` を参照）
- 「3. GROUP BY とウィンドウ関数の使い分け」の前半、「従属」「手順3」
- 集約したメジャーを出す場合の流れ（ディメンション → GROUP BY、メジャー → 集約関数）
- どのテーブルをどのキーで結合するかの整理。従属した属性は JOIN で最後に取得すればよい
- COUNT、SUM、AVG の役割と GROUP BY との組み合わせ
- 複数テーブルを LEFT JOIN して集計するパターン

### ハンズオン
- 演習4：GROUP BY / HAVING（4-1, 4-2）
- 演習5：JOIN（5-1, 5-2）
- 演習6・7：SUM 関数の基本（6-1, 6-2, 7-1, 7-2）※DATE_TRUNC による月別集計
- 演習8：SUM + 集約関数の応用（8-1, 8-2）
- 演習4〜8のうち、苦手だったものを再度解く

**ポイント**: 「このクエリのディメンション・メジャー・フィルター条件」を言語化する練習をする。

## 第3週：ウィンドウ関数と条件分岐

### 目標
- ウィンドウ関数（SUM OVER）で累積集計が書けるようになる
- CASE で条件分岐したラベル付けができるようになる
- GROUP BY とウィンドウ関数の使い分けを理解する

### 座学（`SQL実装の言語化.md` を参照）
- 「3. GROUP BY とウィンドウ関数の使い分け」全体
- GROUP BY：行が集約される / ウィンドウ関数：行はそのまま残り集計値が付く
- PARTITION BY と ORDER BY の役割
- CASE で条件に応じた値を返す。集計と組み合わせると条件付き集計も可能
- LEFT JOIN + WHERE IS NULL で「存在しない」レコードを探すパターン
- サブクエリと JOIN の使い分け、COALESCE で NULL を扱う

### ハンズオン
- 演習9：ウィンドウ関数（9-1, 9-2）
- 演習10：CASE / 条件分岐（10-1, 10-2）
- 演習11・12：LEFT JOIN と複雑な集計（11-1, 12-1, 12-2）
- 演習13：サブクエリ（13-1, 13-2）
- 演習9〜13を振り返り、苦手だったものを再度解く

**ポイント**: 「このクエリは GROUP BY とウィンドウ関数のどちらが適切か？」を判断する練習をする。

## 第4週：統合と実践

### 目標
- 全体の流れを自分のものにする
- チェックリストに沿って、ゼロからクエリを組み立てられるようになる

### 座学（`SQL実装の言語化.md` を参照）
- 「4. 定義に基づいてSQLを書く」「5. まとめ：チェックリスト」
- チェックリストを紙に書き出して手元に置く
- 複数ディメンションを組み合わせた集計の考え方
- つまずいた箇所の確認。自分用の「よく使うパターン」メモを作成する

### ハンズオン
- 設計問題：「コースごとのアクティブ受講者数と、そのコースの平均レビュー評価」を出すクエリを、チェックリストに沿って設計してから書く
- `sql-practice.md` の「追加アイデア（発展）」から1〜2つ選んで実装する
  - 例：月別・コース別の売上ヒートマップ、簡易コホート分析、レビュー評価の平均と標準偏差の比較
- 演習1〜13を、解答を見ずに一通り解き直す

## 次の1ヶ月の予定

この1ヶ月で基礎と実践をマスターしたあと、**次の1ヶ月**では **BigQuery** を使ったより高度なクエリに取り組みます。

**内容**
- BigQuery の Google アナリティクス サンプルデータを使った実践演習
- 日次セッション数、デバイス別ユーザー分析、流入元別の成果
- **UNNEST** を使ったヒットデータの展開（配列・ネスト構造の扱い）
- 広告経由の CVR、ファネル分析、商品別売上ランキング
- ウィンドウ関数を使った「初回訪問から2回目訪問までの期間」分析
- 商品ごとのカゴ落ち率、離脱率の高いページ、LTV によるランク分け など

実務に近い Web 解析データを扱いながら、BigQuery 固有の構文や大規模データ分析の考え方を学んでいきます。
